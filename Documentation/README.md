# Documentation

`Documentation` 是项目总文档目录。

## 这个文件夹干什么？

保存需求、架构、教程、渲染管线、测试计划和术语说明。

## 这里应该放什么代码或内容？

这里不放 C++ 实现代码。这里放项目级 Markdown 文档，用来指导后续实现。

## 文档列表

- `Requirements.md`：完整功能需求和验收标准。
- `Architecture.md`：整体架构和模块依赖。
- `Tutorial.md`：从 0 到 v0.1 的实现路线。
- `RenderPipeline.md`：现代渲染管线规划。
- `QA.md`：测试和验收计划。
- `Glossary.md`：术语表。

## 它依赖谁？

文档不依赖代码，但需要和代码结构保持同步。

## 完成到什么程度算合格？

- 每个核心模块都有对应说明。
- 每个 v0.1 功能都有验收标准。
- 文档能指导下一步应该创建什么文件和写什么类型代码。
